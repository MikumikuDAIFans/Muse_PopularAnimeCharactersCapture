# Muse DataLoad PostgreSQL 迁移与执行指南

Last updated: 2026-04-27 16:55 Asia/Shanghai

本文用于按“近两年全站元数据 JSONL -> PostgreSQL -> 官方角色榜单 -> 榜单快照 -> 定向下载任务”的路线执行。全站阶段不要下载图片。

## 0. 执行原则

- PostgreSQL 是唯一正式数据库。
- JSONL 是原始真源，PostgreSQL 是分析、聚合、榜单和任务状态主库。
- 图片下载不属于全站同步流程，只能由榜单角色或榜单快照驱动。
- 主跑按 monthly 分片，失败后只补失败片。
- manifest 必须固化到 `sync_job / sync_shard / job_log`，避免只依赖文件状态。

## 1. 环境准备

安装：

- Git
- Python 3.12 或更新版本
- Node.js LTS
- PostgreSQL 16+
- 可选 Docker / Docker Compose

创建 Python 环境：

```powershell
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install --upgrade pip
.\.venv\Scripts\pip.exe install -r backend\requirements.txt
```

配置 `.env`：

```powershell
@"
DANBOORU_LOGIN=
DANBOORU_API_KEY=
CRAWLER_RPS=1.0
OUTPUT_ROOT=./output
DATABASE_URL=postgresql+asyncpg://muse_user:password@127.0.0.1:5432/muse_dataload
"@ | Set-Content -Encoding UTF8 .env
```


## 14. 故障处理

- 429 或 5xx：降低 `CRAWLER_RPS`，然后重跑同一条 `--resume` 命令。
- 单个分片异常：优先用 `scripts/retry_failed_shards.py` 按 manifest 补跑。
- JSONL 有重复 ID：manifest 会记录 `duplicate_ids`；PostgreSQL 导入按 post id upsert。
- 磁盘不足：先保留 JSONL，延后导入 PostgreSQL；必要时使用季度分片或外接硬盘作为 `--output-root`。
- Docker API 返回 `/app/output/...`：这是容器内路径，宿主机对应 `output/...`。
- 代理中断：先恢复代理，再执行失败片补跑；不要直接整轮重跑。
