## 当前目标
Muse DataLoad 已升级为 PostgreSQL-only 的正式数据系统。当前本地开发区目标是确保“JSONL -> PostgreSQL -> 官方综合榜/新兴榜 -> 月度聚合 -> 榜单快照 -> 下载任务骨架”完整跑通，并等待 SSH 后在远端执行 24 个月正式数据上线运行。

## 范围与约束
- In scope:
  - PostgreSQL 作为唯一正式数据库运行栈。
  - 官方数据库链路生成综合热门榜和年龄过滤新兴榜。
  - 任务状态表、月度聚合层、榜单快照层、下载任务层。
  - 本地真实 PostgreSQL 中规模端到端验证。
- Out of scope:
  - 未经用户明确指令，不执行任何图片下载。
  - 不把临时 JSONL 直出榜单作为正式榜单口径。
  - 本轮不引入 ClickHouse、队列平台或完整多机集群。
- Locked decisions:
  - PostgreSQL 是唯一正式数据库技术栈。
  - JSONL 保留为原始真源，PostgreSQL 是分析与任务状态主库。
  - 下载层只接受榜单快照或榜单角色输入，不提供全站图片下载默认路径。

## 已完成改造
### Phase 1: 官方链路修复与基线固化
- `backend/services/ranking.py` 已修复大库综合榜聚合路径，并写入 `ranking_snapshot / ranking_snapshot_item`。

- `backend/requirements.txt` 使用 `asyncpg` 与 `psycopg[binary]`。
- `scripts/import_jsonl_fast.py` 已升级为 PostgreSQL JSONL 导入器。
- `scripts/recount_tag_post_counts.py` 已升级为 PostgreSQL tag 计数回填工具。
- `scripts/sync_manifest_to_db.py` 可把 manifest 固化为任务状态记录。

### Phase 3: 聚合层与快照层

  - `ranking_snapshot_item`
- `backend/services/aggregation.py` 与 `scripts/rebuild_character_monthly_stats.py` 已完成 PostgreSQL 月度聚合重建。

### Phase 4: 下载层分离
