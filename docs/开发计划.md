## 目标

Muse DataLoad 的目标是基于 Danbooru 内部数据，完成“角色榜单发现 -> 榜单校验 -> 角色样本导出 -> 前端/CLI/API 服务化”的闭环：

- 生成最近半年综合热门角色 Top 200 榜单，产出 JSON/CSV。
- 生成独立的新兴热门角色榜，排除古早常青角色，保留近期活跃且年龄较小的角色。
- 支持按角色导出训练样本目录，包含图片、txt caption、json 元数据三件套。
- 支持 Docker 后端、前端、CLI、脚本化校验和本地 SQLite 持久化。

## 范围与约束

- 数据来源只使用 Danbooru 内部 API 与本地缓存数据库，不引入外部榜单或人工热度源。
- 角色候选只接受 Danbooru `category=character` tag。
- 作品信息来自本地帖子 tag 共现与 copyright tag 聚合。
- 新兴角色“年龄”口径为：当前时间减去 `first_seen_at`，其中 `first_seen_at` 来自 Danbooru 最早帖子查询并缓存。
- 默认数据窗口为最近 6 个月，默认综合榜 `top_n=200`、`min_count=50`，默认新兴榜 `top_n=200`、`min_count=50`、`min_recent_count=10`、`max_age_days=1095`。
- 当前数据库为本地 SQLite：`muse_dataload.db`。Docker 通过 volume 挂载数据库和 `output/`，可重启保留状态。

## 关键实现

### 数据采集与入库

- `backend/services/danbooru.py`：Danbooru API 客户端、tag/post 查询、alias/implication 同步基础能力。
- `backend/services/ingest.py`：帖子与 tag 入库。
- `scripts/sync_recent_posts.py`：最近窗口帖子同步，使用 Danbooru `date:YYYY-MM-DD..` 搜索。
- `scripts/import_jsonl_fast.py`：本地 JSONL 快速导入，用于大批量元数据落库。
- `scripts/sync_tags.py`：character/copyright tag 字典同步。

### 综合角色榜

- `backend/services/ranking.py`：综合热门角色榜生成，按本地最近窗口数据、tag 总帖数、recent count 与 copyright 共现聚合计算。
- `scripts/build_character_list.py`：生成正式综合榜 JSON/CSV。
- `scripts/validate_character_export.py`：校验 JSON/CSV 数量、rank、字段与一致性。
- `scripts/audit_copyright_coverage.py`：审计 copyright 覆盖率。
- `scripts/analyze_thresholds.py`：阈值分析。

综合榜字段包括：

- `rank`
- `character_tag`
- `copyrights`
- `post_count`
- `recent_post_count`
- `popularity_score`
- `stat_at` / `generated_at`

### 新兴角色榜

- `scripts/enrich_character_birthdates.py`：补齐并缓存 `first_seen_at`、`first_seen_post_id`。
- `backend/services/emerging.py`：新兴榜生成、当前年龄重算、`recent_ratio` 与 `growth_score` 计算。
- `scripts/build_emerging_character_list.py`：生成独立新兴榜 JSON/CSV。
- `scripts/validate_emerging_export.py`：校验字段、rank、年龄阈值与 JSON/CSV 一致性。
- `backend/api/routes.py`：`GET /api/characters/emerging` 和 `POST /api/characters/build-emerging`。
- `frontend/src/pages/DataPage.vue`：新兴榜前端查询、刷新、导出、按行样本导出。

新兴榜字段包括：

- `rank`
- `character_tag`
- `copyrights`
- `post_count`
- `recent_post_count`
- `first_seen_at`
- `character_age_days`
- `recent_ratio`
- `growth_score`
- `birth_confidence`
- `lifecycle_notes`

新兴榜与综合榜保持独立输出。综合榜不因新兴榜过滤规则被破坏。

### 训练样本导出

- `backend/services/dataset.py`：按角色导出样本目录，生成图片、txt caption、json 元数据。
- `backend/api/routes.py`：`POST /api/datasets/export`。
- `scripts/export_sample_dataset.py`：脚本化样本导出。
- `scripts/validate_dataset.py`：校验每个样本的图片/txt/json 三件套、caption 一致性、关键 JSON 字段。

目录结构：

```text
output/dataset/<safe_character_tag>/<post_id>.<ext>
output/dataset/<safe_character_tag>/<post_id>.txt
output/dataset/<safe_character_tag>/<post_id>.json
```

Caption 顺序：

```text
subject -> character -> copyright -> artist -> general -> other
```

### 服务化与前端

- `docker-compose.yml`：后端、前端、SQLite 与 output volume 挂载。
- `backend/api/routes.py`：任务、帖子、tag、榜单、导出、统计 API。
- `frontend/src/pages/DataPage.vue`：帖子、标签、综合榜、新兴榜与统一样本导出弹窗。
- `scripts/smoke_api.py`：后端 API smoke。
- `cli/main.py`：CLI 操作入口。

## 关键决策

- 保留综合榜 `character_list_recent_6m_top_200.*` 作为主榜，不被新兴榜逻辑改变。
- 新兴榜单独输出 `character_list_emerging_6m_top_200.*`，使用 `growth_score` 排序。
- 新兴榜年龄动态口径为“当前时间 - 首次出现”，前端和后端缓存读取均重算当前年龄。
- 新兴榜前端只保留“刷新”，不暴露“重建”按钮，避免用户误解刷新与重新构建的边界。
- 综合榜和新兴榜都按行打开同一个样本导出弹窗，避免全局导出控件造成目标角色不明确。
- 图片下载保持 Danbooru 友好策略：原图失败时尝试 large/sample/preview URL；仍失败则记录错误并跳过该样本。
