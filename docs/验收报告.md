# 性能优化与自动榜单更新阶段性验收

## 验收结论

阶段性通过。已完成代码层升级和本地 PostgreSQL 单元/集成级验证，具备批量导入、增量月度聚合、每周榜单更新入口、dry-run、窗口水位线、任务状态和运维文档。24 个月正式数据性能基线和真实每周小窗口端到端联网同步仍待执行。

## 本轮实现内容

- `scripts/import_jsonl_fast.py`
  - 从逐行 upsert/delete/insert 改为批量 post upsert、批量 tag upsert、批量 post_tag 删除/插入。
  - tag 计数回填改为集合 SQL 更新。
  - 同批重复 post id 以最后一条为准，避免 PostgreSQL 同语句 upsert 冲突。
  - 当传入的旧 `task_id` 不存在于 `task` 表时，自动写入 `NULL`，避免与正式 `sync_job/sync_shard` 状态体系冲突。
- `backend/services/aggregation.py`
  - `rebuild_character_monthly_aggregates()` 新增 `start_date` / `end_date` 参数。
  - 不传日期时保留全量重建；传日期时只刷新受影响月份。
- `backend/services/ranking.py`
  - 综合榜在月度聚合存在时优先读取 `character_monthly_stats` 和 `character_monthly_copyright`。
  - 无聚合数据时保留原 `post_tag` 精确扫描回退路径。
- `backend/services/emerging.py`
  - 新兴榜在月度聚合存在时优先读取 `character_monthly_stats` 和 `character_monthly_copyright`。
  - 无聚合数据时保留原 `post_tag` 精确扫描回退路径。
- `scripts/weekly_update_rankings.py`
  - 新增每周增量更新入口。
  - 支持 `--dry-run`、`--start-date`、`--end-date`、`--lookback-days`、`--resume`、`--database-url`、`--output-root`、`--job-key`。
  - 每次运行生成窗口化 job key，例如 `weekly-ranking-update:2026-05-08:2026-05-15`。
  - 串联元数据同步、JSONL 导入、manifest 固化、增量聚合、综合榜、新兴榜、导出校验和任务状态写入。
- `scripts/collect_performance_baseline.py`
  - 新增只读性能基线采集入口。
  - 记录核心表行数、post 时间范围、最近榜单快照。
  - 支持 `--explain` 输出关键查询的 `EXPLAIN (ANALYZE, BUFFERS)`。
- 文档
  - `README.md` 新增每周增量更新命令。
  - `docs/迁移与执行指南.md` 新增调度、查看任务、查看 snapshot、失败恢复和补跑说明。
  - `docs/开发计划.md` 与 `docs/验收清单.md` 已升级为本轮任务书和验收标准。
