## 验收核对

### 综合榜

综合榜已满足自动化验收条件：

- 使用 Danbooru 内部数据。
- 使用 `category=character` tag 作为候选。
- 输出 JSON/CSV Top 200。
- 字段包含 rank、character tag、copyrights、post count、recent count、popularity score。
- JSON/CSV 一致性校验通过。
- copyright 覆盖率与 Top 200 自动抽检已有报告产物。

### 新兴榜

新兴榜已满足验收条件：

- 独立于综合榜输出。
- 包含 `first_seen_at`、`first_seen_post_id`、`character_age_days`、`recent_ratio`、`growth_score`。
- 年龄使用当前时间动态计算。
- 默认阈值下产出 128 条真实数据。
- API、前端代理与前端页面均接入。
- 前端只保留“刷新”，并支持按行样本导出。

### 训练样本导出

训练样本导出已满足小批量真实数据验收条件：

- 目录结构稳定：`output/dataset/<character>/<post_id>.*`。
- 每条样本包含图片、txt caption、json 元数据。
- json 保留原始 tag、tag groups、caption、score、fav_count 等核心字段。
- caption 顺序满足 v0.1。
- 下载成功、跳过、失败均有记录路径。
- 原图 403 fallback 已经通过银狼样本和随机多角色回归验证。

### 服务化

核心服务化验收通过：

- Docker 后端 healthy。
- 前端 nginx running。
- `/api` 代理可用。
- SQLite 持久化可用。

- 前端生产构建通过。

## Review 结论

未发现阻塞交付的问题。

需要在后续迭代继续关注的点：

- Docker API 返回的 `dataset_dir` 是容器内路径，例如 `/app/output/dataset/...`；宿主机实际路径为 `output/dataset/...`。
- WebSocket 实时日志前端消费仍可增强，但不影响榜单与样本导出主流程。
- 100 万帖子级 SQLite 查询压测尚未执行；当前已验证数据规模为 26,400 posts。
- 新兴榜 `first_seen_at` 依赖 Danbooru 最早帖子与历史 tag 状态，建议业务侧人工抽样复核前 20 名。

## 最终判断

本项目已经从开发、验证、真实导出回归和文档状态四个层面完成正式收尾。当前可以作为 Muse DataLoad 的最终可交付版本进入业务复核或后续增量迭代。
